# 口播稿：开源 GitHub 卡片视频 Skill

## 原版

我最近一直在试一件事：用 AI 做视频。

一开始是因为我在网上看到很多人用 Remotion、HyperFrames 这类工具做视频，有些开场、转场和信息动画确实挺好看。

我自己也想试试。

但真折腾起来就发现，想做出那种完整的视频效果，并不是一句提示词就能解决。

开场怎么设计，信息怎么排，截图放哪里，封面怎么生成，最后怎么合成，里面有很多细碎步骤。

所以我先做了一个简单一点、但能稳定产出的东西。

它是一个 Codex Skill，名字叫 github-repo-video-card。

作用很直接：给它一个 GitHub 仓库，它把这个开源项目整理成一条中文卡片视频。

如果我没有想好今天做什么，也可以让它先找选题。

它会先给几个候选项目，说明这个项目是干嘛的，页面素材够不够，适不适合做短视频。

我确认之后，它再去读 README、整理项目用途、规划截图、写中文说明、做分镜、准备封面提示词、标题和标签。

如果本地工具链接好了，还可以继续用 HyperFrames 合成视频。

这套东西技术含量不算多高，但它把每天重复的步骤收在一起了。

我现在先把它开源出来。

里面不是一个神秘工具，也不是保证你一点就出大片。

它更像是一个可以改的 AI 视频工作流样板。

你可以拿去改成自己的选题规则、文案风格、封面风格，或者只用它的文字部分。

我也会继续用它每天做 GitHub 开源项目卡片视频。

我自己也想把视频效果继续往上提。

如果你也在用 AI 做视频，或者你有自己的 Remotion、HyperFrames、Codex Skill 工作流，也欢迎分享给我。

我开源这个东西，也有一部分原因是想看看别人怎么做，互相抄作业。

如果你对这种 AI 做视频的流程感兴趣，可以去 GitHub 看一下。

跑不通也正常，评论区说一声，我后面再把安装和使用过程单独录一条。
